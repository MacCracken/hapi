# hapi 1.0.0 — release notes

**Date:** 2026-05-23
**Tag:** `1.0.0` (bare semver per project convention — no `v` prefix)
**Genesis:** [agnosticos](https://github.com/MacCracken/agnosticos)

hapi 1.0.0 is the first **frozen-contract** release. Everything that
shipped at v0.9.0 — ten verbs, three global flags, the `hapi.cyml`
manifest schema, and the JSONL audit-trail format — becomes
contractual at this tag.

## What "v1.0" actually means

Three surfaces freeze:

1. **Command surface.** Every verb's argument shape, exit codes,
   and stdout / stderr line shape is the v1.0 contract. New verbs
   may land in v1.x as additive features; existing verbs may not
   change shape without a v2 major.
2. **`hapi.cyml` manifest schema.** ADR 0001 is the contract.
   `[package]` + `[[link]]` rows + `ignore` glob list. Manifest
   files written today will be readable by every v1.x release.
3. **Audit-trail format.** ADR 0002 is the contract — JSONL,
   the required field set, `sha1c:` canonical-hash prefix, the
   additive-growth rule. Trails written by v1.0.0 stay readable
   indefinitely; pre-v1.0 trails with the `sha1:` raw-bytes
   prefix continue to be tolerated by readers per the migration
   contract.

All five ADRs (manifest schema / audit-trail format / symlink
target shape / adopt semantics / capability-bounded roots) carry
**Status: Frozen at v1.0.0 (2026-05-23)** as of this release.

## The verb set frozen

| verb              | summary                                                | flags                   |
|-------------------|--------------------------------------------------------|-------------------------|
| `hapi inspect`    | parse + print a manifest                               |                         |
| `hapi link`       | create the package's symlinks                          | `--force`               |
| `hapi unlink`     | remove the package's symlinks (audit-trail-driven)     |                         |
| `hapi adopt`      | move a file into a package, link back                  |                         |
| `hapi rollback`   | reverse the audit trail to the last marker             |                         |
| `hapi checkpoint` | append a rollback-marker                               |                         |
| `hapi status`     | report drift between manifest and on-disk symlinks     |                         |
| `hapi list`       | enumerate packages tracked in the trail                |                         |
| `hapi check`      | parse + display, optionally strict                     | `--strict`              |
| `hapi sync`       | re-apply a manifest (idempotent)                       |                         |

Plus three global flags (apply to a subset of verbs each):

- `--root <path>` (link / adopt / sync / status / list) — opens a
  non-`$HOME` scope, gated by the capability check.
- `--dry-run` (link / unlink / adopt / sync / rollback /
  checkpoint) — print the plan; write nothing.
- `--backup-to <dir>` (link / sync / adopt) — opt-in pre-`--force`
  snapshot of regular-file conflicts.

Plus `--version` / `-v` / `--help` / `-h` on the dispatcher.

## What ships in 1.0.0 vs prior versions

1.0.0 is a **promotion**, not a feature cut. The code that ships
the tag is the v0.9.0 binary with the contract-freeze
announcement. No new functionality.

For the feature history of how we got here, see the
[CHANGELOG](../../../CHANGELOG.md) — every v0.x release
documented its scope, breaking changes, and rationale.

## What's deferred post-v1.0

Captured in [`docs/development/roadmap.md`](../roadmap.md) under
*Deferred during prior milestones (post-v1.0 candidates)*. None
block the v1.0 surface; each carries its own rationale and the
ADR or issue file that documents the deferral. Highlights:

- **`hapi sync --prune`** — remove trail rows no longer in the
  manifest. Today's full-rotation workaround (`unlink + edit +
  link`) ships at v1.0; the `--prune` surface may land
  default-on at a major bump if the dogfood pressure earns it.
- **`hapi status --quiet`** — flip the exit-code semantic for
  scripted `&&` chains. The v1.0 fix is the guide-doc note;
  the flag is post-v1.0 if the pattern recurs.
- **`hapi merge <pkg>`** — speculative 3-way merge driver for
  drifting upstream templates (hyprland-class). Needs a
  stock-template-version data model before earning its place.
- **`hapi trail compact`** — the audit trail grows monotonically
  by design; ADR 0002 reserves a compact verb for later.
- **`hapi rollback --to <marker> / --steps N`** — refinement of
  the no-arg walk-to-last-marker semantic that ships at v1.0.
- **Per-package `manifest_path` field on audit entries** — would
  unlock a no-arg `hapi sync` that walks every tracked package
  without a tree-walk heuristic. Additive; not blocking.

## Known upstream dependencies

Tracked in roadmap.md *Upstream dependencies*. None gate v1.0:

- **`sys_rename` / `sys_fsync` / `sys_fdatasync` stdlib wrappers**
  — proposed in cyrius; hapi currently calls `syscall(N, ...)`
  directly with the magic-number cross-arch hazard. When the
  wrappers land, hapi's call sites collapse to named calls in a
  quality-of-life patch.
- **kavach capability API** — replaces the v0.x
  `HAPI_ALLOWED_ROOTS` env-var stopgap inside `src/cap.cyr`.
  Internal refactor only; `cap_check_root_r(path) -> Result`
  signature is frozen at v1.0. Also closes F-002 from the
  security audit.
- **cyriusly starship-install non-clobbering fix** — upstream
  consumer-package bug surfaced during M7 dogfood; when it
  lands the `dotfiles/starship` caveat note can drop.

## Upgrading from v0.9.0

No code changes are required. The v0.9.0 binary parses and writes
the v1.0 format. The v1.0 binary parses every v0.x trail (`sha1:`
prefixes continue to be accepted by readers). Existing manifests
continue to parse.

If you have a v0.x audit trail you want to "graduate" to canonical
hashes, the natural moment is the next `hapi link` / `hapi adopt`
that touches the package — those writes produce `sha1c:` entries
without needing a migration command.
