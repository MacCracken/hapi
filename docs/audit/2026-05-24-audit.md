# 2026-05-24 — P(-1) re-walk (1.0.x-final)

**hapi version**: 1.0.0 (1.0.x patch line; no surface change).
**Auditor**: maintainer pass per CLAUDE.md Process P(-1), scoped
in [`README.md`](README.md) as the 1.0.x-final arc.
**Coverage**: a *re-walk*, not a re-design. The v1.0 contract is
frozen and no verb / path argument / capability surface changed
since [`2026-05-23-audit.md`](2026-05-23-audit.md). This pass
(1) confirms the v1.0 findings still hold, (2) reviews the only
code delta since — syscall-number naming — for any change in
syscall-return handling, and (3) dispositions one new boundary
surfaced by a drive-move dogfood run.

## Scope

The four threat classes from ADR 0005 + CLAUDE.md, re-confirmed
against the unchanged v1.0 surface:

1. **Path traversal** — `..` past the validator or cap-check.
2. **Symlink loops** — readlink chasing during probe / link / adopt.
3. **TOCTOU** — probe-vs-act races.
4. **Capability boundary** — does `--root` + `HAPI_ALLOWED_ROOTS`
   bound writes?

Plus the accumulated items the arc README flagged: F-002's
deferral status, and the state-dir-loss recovery story.

## Methodology

The attack surface is byte-for-byte the v1.0 surface — no new
verbs, flags, or path arguments. The only source change since
2026-05-23 is **syscall-number naming** (bare `syscall(2/3/74/82,
…)` → named `SYS_OPEN` / `SYS_CLOSE` / `HAPI_SYS_FSYNC` /
`HAPI_SYS_RENAME`); the constants resolve to identical numbers, so
codegen is unchanged. Re-confirmed the F-001 / F-003 defenses by
their standing test groups rather than re-deriving them, and
reviewed the renamed call sites for return-handling regressions.

## Findings

### F-001 — `--root` `..` bypass [HIGH] — STILL FIXED

`fsl_lexical_normalize` remains the cap-check's normalizer
(`src/cap.cyr:80` per-component, `:102` / `:110` for the `--root`
and `$HOME` values). The syscall-naming change did not touch
`fs_link.cyr`'s normalize logic. Defense asserted green by the
`audit-repair: lexical normalize collapses ..` and
`audit-repair: cap check rejects .. that escapes $HOME` groups in
the 242-assertion suite.

### F-002 — `--root` symlink escape [MEDIUM] — STILL DEFERRED

Unchanged. `cap_check_root_r` is still lexical-only; per-component
symlink resolution still waits on the kavach capability service,
which has **not** shipped a stable API as of this pass. The
deferral rationale in
[`../development/issues/2026-05-23-cap-check-symlink-escape.md`](../development/issues/2026-05-23-cap-check-symlink-escape.md)
holds; no consumer relies on cap_check resolving symlinks today.

### F-003 — backup-copy TOCTOU symlink race [LOW] — STILL FIXED

`src/backup.cyr` still opens the source with `O_NOFOLLOW`
(`0x20000`). Untouched by this pass. Defense asserted by the
`audit-repair: backup_copy fails on symlink source (TOCTOU
defense)` group.

### F-004 / F-005 — audit-trail trust model, probe-act TOCTOU [INFO] — STILL ACCEPTED

Unchanged boundaries per ADR 0002 / ADR 0004. No code change.

### F-006 — audit trail lost on state-dir wipe [LOW] — ACCEPTED (boundary)

**What.** A drive move / fresh `$XDG_STATE_HOME` wipes
`audit.jsonl`. The relative symlinks survive (they moved with
`$HOME`), and `hapi status` confirms 0 drift, but `hapi list` and
`hapi rollback` undercount: re-running `sync` over already-correct
links is an idempotent no-op that writes no trail entry, so the
wiped trail is **not** rebuilt to match the live link set.

**Demonstrated.** 2026-05-24 drive-move recovery: 24 links live
across 11 packages, `hapi list` reported 2 (the git `.gitconfig`
creation + the zsh `.zshrc` force-relink — the only two ops the
recovery actually performed). Reproduced as a unit test:
`sync: recovers after the audit trail is lost (drive-move
scenario)`.

**Impact.** No data loss and no incorrect links — `status` (the
declarative truth over the manifests) is intact. The divergence is
confined to the *operational log*: `list` undercounts and
`rollback` cannot reverse links that predate the wipe. Severity
Low; bounded to "rollback history resets when local state is lost."

**Disposition — accepted.** Consistent with F-004: the trail is
ephemeral local state by design (ADR 0002; cross-machine sync is
permanent out-of-scope per the roadmap). The honest semantic is
"`status` is the source of truth after a state-dir loss; the trail
starts fresh." A reseed affordance (`sync` re-emitting entries for
already-correct links) is a *possible* additive v1.x ergonomic but
is not required for correctness and would itself need the
no-arg-sync work. Tracked at
[`../development/issues/2026-05-24-audit-trail-lost-on-state-dir-wipe.md`](../development/issues/2026-05-24-audit-trail-lost-on-state-dir-wipe.md).

## Cleanup reviewed (syscall naming)

The bare syscall numbers were named with no behavior change:

| Site | Before | After | Source |
|------|--------|-------|--------|
| probe open / close | `syscall(2/3, …)` | `SYS_OPEN` / `SYS_CLOSE` | stdlib `syscalls_x86_64_linux.cyr` |
| atomic write fsync | `syscall(74, …)` | `HAPI_SYS_FSYNC` | `HapiSysno` enum (manifest_write.cyr) |
| rename (×5) | `syscall(82, …)` | `HAPI_SYS_RENAME` | same enum |

Return handling at every renamed site is unchanged: `fd < 0` on
open, `rrc < 0` on rename, `gd >= 0` on getdents. The `HapiSysno`
enum carries a removal marker pointing at the cyrius `sys_rename` /
`sys_fsync` proposals — the coordinated-swap point for the
roadmap's stdlib-wrapper item. `HAPI_SYS_` prefix avoids colliding
with the future stdlib `SYS_RENAME` / `SYS_FSYNC`.

DCE: `CYRIUS_DCE=1` produces a byte-identical binary (226 056
bytes), so the build-time "310 unreachable fns" note is purely
informational — the linker already drops them. No action.

## Sign-off

The v1.0 security posture is unchanged: F-001 and F-003 remain
fixed and test-asserted, F-002 remains the documented kavach-time
follow-up, F-004 / F-005 remain accepted boundaries. The 1.0.x
syscall-naming cleanup is behavior-neutral. The one new finding,
F-006, is an accepted boundary documenting that the audit trail is
ephemeral local state. This closes the 1.0.x-final P(-1) arc — the
next pass is before a v2.0 cut or on the next change to a path
argument, syscall, capability surface, or audit-trail format.
